QuiVo

Learning vocabulary quickly on-the-go

ROLE

Researcher
UX Designer

Duration

June - July 2020

TOOLS

Invision, POP

“QuiVo” is a vocabulary app that I created during my CareerFoundry intro course. This project allowed me to implement design principles to create a vocabulary app. From research to usability testing, the main focus of this project was to understand and utilize the design process.

Objective

To create a vocabulary/language learning app that not only provided general knowledge but that is also customizable to users

Competitive Analysis

SWOT Analysis was conducted on three vocabulary/language learning apps.

Drops

Quizlet

Vocabulary

Summary:

  • Providing users with the option to add information allows them to contribute to their learning.

  • Good UI design is important but focus should be on the content to not distract from main purpose of using the app.

  • Navigation of the app should be intuitive without any friction to help with finding information quickly and easily.

  • Variability is needed in the learning features to keep users engaged, allowing them to enjoy the learning process.

Usability Test

Usability testing was carried out with five participants (ages ranging from 24 – 28) using POP prototype on phones/laptops. Participants were asked to complete three tasks. An adapted version of Jakob Nielsen’s usability rating scale was used to evaluate errors.

Tasks

1. Sign up as new user

2. Add new vocabulary

3. Review a category of flashcards

Usability Rating Scale

0 = I don't agree that this is a usability problem at all

1 = Cosmetic problem only: need not be fixed unless extra time is available on project

2 = Minor usability problem: fixing this should be given low priority

 3 = Major usability problem: important to fix, so should be given high priority

 4 = Usability catastrophe: imperative to fix this before product can be released

Iterations

Navigation

With navigation, the main issue that resulted in a severity of 4 was the difficulty of finding certain sections.

Labeled sections to make it easier for user to navigate the pages

Adding Vocabulary

There was confusion about the "Add button". A participant also mentioned concern about the accuracy of pronunication for the recording feature. These resulted in a usability rating of 4. A point of friction was also observed by a participant about finding a word on the list quickly, resulting in a severity of 3.

Changed audio feature provides correct pronunciation sourced from Internet; options to record when unavailable

Search bar added to help users quickly look up vocabulary

"Add" button moved below to reduce confusion

Reviewing Vocabulary

Participants preferred to have an indication of completion when reviewing the flashcards and to swipe instead of clicking arrows. Though these issues resulted in a severity of 2, changes were considered.

Swipe instead of clicking arrows to change between flashcards

Users given indiciation of flashcard deck completion

What I learned...

Working on this project allowed me to put into practice the principles of the design process. I realized the importance of listening to users and empathizing with them. Their feedback is valuable and crucial in improving my designs. However, I also learned that it is impractical to solve every problem. It is important to prioritize errors to discover the best solutions to create successful designs.

What's next?

The next step would be to create a high –fidelity prototype, incorporating the iterations made on the designs. More layouts could be designed for other features of the app as well. There are definitely improvements to be made on the UI design front to make the app more engaging to users. Further usability testing should also be conducted using the prototype and continuous iterations, if necessary.
